## Why this review exists
§1.5 is the compliance linchpin: it gates what enters memory, who can recall it, and how a data subject
is erased — directly touching APP 1/3/5/6/11/11.2/13 and anti-discrimination law. A regulated, local-first
HR product reviews this component closely. This covers the governance package + its wiring into the
curated-write and candidate-ingest paths and the model-facing tool.

## Checklist

| # | Concern | Finding |
|---|---|---|
| 1 | **SQL injection (audit log)** | `audit.py` uses **parameterised** `?` placeholders for every INSERT/SELECT; the `WHERE` clause is built from a fixed allow-list of columns (`target_key`/`action`), values bound as params. No string-built SQL. ✅ |
| 2 | **Write path cannot be bypassed by the model** | The governed `memory` tool reaches the model only via the `ToolRegistry`→`manager.handle_tool_call` bridge; the handler runs `GovernanceGate.validate` **before** any `store.add/replace`. Unknown `action` is rejected (not defaulted to `add`); unknown `target` rejected. 100% of *accepted* curated writes carry the rendered governance header. ✅ |
| 3 | **Candidate (PII) ingest is governed** | `CandidateMemoryProvider.ingest` runs `GovernanceGate.validate_entity_ingest` when a gate is injected: missing provenance (`source_refs`) → `rejected:no_provenance`; `consent_status ≠ "granted"` → `rejected:no_consent`; both audited as `write:ingest`. Closes the post-review gap where candidate PII could be written unlabelled. Bias hygiene is deliberately **not** run on candidate content (rejecting a résumé for naming a protected attribute would itself discriminate). ✅ |
| 4 | **Residual ungoverned write surface** | `MemoryStore.add/replace/remove` remain public and the builtin still exposes `.store`, so a *programmatic* caller could write around the gate. Accepted under YAGNI **and documented**: the governed provider handler is the sole writer to the curated store, and the Plan §1.5 + §1.6 mandate that any future programmatic writer (e.g. pre-compression persist) route through the gate. Not a model-reachable path. ⚠️ (documented invariant) |
| 5 | **Bias-scanner correctness / ReDoS** | Matching is **word-boundary** regex, not substring — fixes the triple-review MAJOR where `"age" ∈ "management"` hard-rejected ordinary HR text. Patterns are simple alternations/stems with bounded `.{0,40}` quantifiers (no nested unbounded repetition → no catastrophic backtracking). Honest-boundary documented: a heuristic, not a complete classifier. ✅ |
| 6 | **Governance metadata leak into the model** | The in-entry header (carrying `consent_id`, `collected_by`, `source_ref`) is stripped from the model-facing snapshot by `strip_governance_headers` (wired in `MemoryBackend.memory_snapshot`); the header stays on disk for validation/back-link. Closes the triple-review MAJOR (M1) where the header would re-enter the frozen system prompt. ✅ |
| 7 | **RBAC recall least-privilege + no-leak** | `scope_predicate` matches a key only when it equals an allowed scope or is nested under `scope + ":"`, so `acme:apac` does **not** authorise `acme:apacX`. The predicate is applied **before** NN scoring/truncation (filter-before-NN, §1.4 seam), so an out-of-scope subject's existence never leaks. `FULL_ACCESS` default is explicit and only for the single-user MVP. ✅ |
| 8 | **Erasure completeness + forensic trail** | `Eraser.erase` cascades structured + vector deletion (exact-or-nested-prefix), clears recall caches, registers backup-ageing, and audits `erase`. A failed `deleter` is audited (`rejected:error`) and re-raised (no silent loss of the attempt); a cache-clear failure downgrades the result to `ok:cache_warn` in the trail rather than reporting a clean erase while recall could still surface the subject. ✅ |
| 9 | **Append-only audit integrity** | `AuditLog` exposes no update/delete method (append-only by construction). Dual timestamp (monotonic + wall) per row. NOTE: this is a tamper-*evidence-light* local log (the underlying SQLite file could be edited out-of-band); cryptographic chaining / WORM is a §1.8 canonical-table concern, not promised here. ⚠️ (scope-appropriate) |
| 10 | **Thread safety** | `AuditLog` opens `check_same_thread=False` with a `threading.Lock` around record/query, so when §1.8's read-path audit records from the §1.3 background `mem-sync` worker it will not raise. §1.5's own audit writes are main-thread only. ✅ (forward-safe) |
| 11 | **No dynamic code / unsafe deserialisation** | No `eval`/`exec`/`pickle`/`yaml`/`subprocess`. Header parse is line-splitting; bias scan is `re`; audit is parameterised SQLite. ✅ |
| 12 | **Transitive dependencies** | **Stdlib only** (`sqlite3`, `re`, `threading`, `time`, `datetime`, `dataclasses`, `typing`). No third-party import added. ✅ |
| 13 | **Honest compliance boundaries (no overclaim)** | Erasure = live-store delete + cache + backup-ageing register, NOT GDPR-instant wipe; de-id of residual mentions → §1.11. Read/recall audit → §1.8. Bias scanner = heuristic. RBAC auth source = injected/deferred. All stated in code + Plan + spec; the triple-review confirmed no overclaim survives. ✅ |
| 14 | **No `agent_loop.py` / `MemoryStore` change** | git-verified: `core/` byte-identical to `main`; the ported `MemoryStore` write methods byte-unchanged. Governance sits in front via the provider write path + the registry bridge. ✅ |

## Conclusion
**Cleared for §1.5 scope.** The write path (curated tool + candidate ingest) is gated and cannot be
bypassed by the model; the bias scanner no longer false-rejects benign text; the governance header does
not leak into the model context; RBAC enforces least-privilege recall with no existence leak; erasure
cascades with a robust forensic trail; the audit log is append-only and thread-safe for §1.8. Tracked
dependencies that complete the posture: read/recall audit + cryptographic tamper-evidence (§1.8), the
real content threat scan (§1.6), residual de-identification (§1.11), the real RBAC auth source
(§1.9 / PRD open-Q#8), and the real embedder/backend (config / §1.12). Documented residual: programmatic
`MemoryStore` writers must route through the gate (not a model-reachable path).
