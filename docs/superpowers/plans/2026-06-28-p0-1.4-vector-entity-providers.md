# §1.4 Vector store + Candidate/Semantic providers Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans (single-agent inline, `CLAUDE.md` §5). Steps use checkbox (`- [ ]`).

**Goal:** Build §1.4's large-volume retrieval memory layer behind the §1.3 `MemoryProvider` contract — vector store, structured store, embedding seam, Semantic + Candidate providers, a minimal Composite, a re-embed migration tool, and a recall+P95 benchmark — all stdlib-only, offline.

**Architecture:** New code under `memory/vector/`, `memory/structured.py`, `memory/embedding.py`, `memory/providers/{semantic,candidate,composite}.py`, `memory/benchmark.py`. Heavy backends (real vector DB, real embedder) and governance/RBAC stay behind injected seams. Attaches to the loop via the §1.3 Composite-as-sole-external + the §1.3 hooks — no `agent_loop.py` change.

**Tech Stack:** Python ≥3.11, stdlib (`sqlite3`, `json`, `math`, `hashlib`, `uuid`, `dataclasses`), pytest. No new deps.

## Global Constraints
- **New code, design-derived** (not a Hermes file port) — add a row to `THIRD_PARTY_NOTICES.md` noting it attaches to the §1.3 ported contract; cite §1.4 in commits.
- **Bilingual docstrings** (EN then 中文) on every file/class/function incl. tests/`__init__.py`; never `"""` inside a docstring.
- **Seams stay injected + default-safe:** `EmbedFn` (default hashing), `write_gate` (pass-through), `scope_filter` (open), `VectorStore` backend (SQLite). Real impls = §1.5 / §1.12 / config.
- **Filter before NN** (no retrieve-then-filter leak). **Drift guard:** no mixed `embed_version` retrieval.
- Offline only (fake embedder, tmp dirs); never call real OpenAI. Run from `agent/`: `python -m pytest`. Commit per layer.

### Task 1: Embedding seam (`memory/embedding.py`)
**Produces:** `EmbedFn = Callable[[str], list[float]]`; `hashing_embedder(dim=256) -> EmbedFn`; `embed_version(name, dim) -> str`; `cosine(a, b) -> float`.
- [ ] **Test** `test_embedding.py`: `e = hashing_embedder(64)`; `cosine(e("Python engineer"), e("python developer")) > 0` (shared "python" dim); `cosine(e("python"), e("gardening leave")) == 0` or near 0; deterministic (`e("x") == e("x")`); vectors L2-normalised (`abs(sum(v*v)-1) < 1e-6` for non-empty); `embed_version("hash", 64) == "hash@64"`.
- [ ] Run → fail; implement (tokenise `re.findall(r"[a-z0-9]+", text.lower())`, hash `int(hashlib.blake2b(tok, digest_size=8).hexdigest(),16) % dim`, count, L2-normalise; empty → zero vec); run → pass; commit `feat(memory): embedding seam (hashing bag-of-words default + cosine)`.

### Task 2: Vector record + store (`memory/vector/`)
**Produces:** `VectorRecord` (dataclass, spec §4); `VectorStore` ABC (`add`/`delete`/`delete_by_key_prefix`/`search`/`current_version`); `SqliteVectorStore(db_path)`.
- [ ] **Test** `test_vector_store.py`: build records with `hashing_embedder`; `add` 3 candidate chunks under keys `acme:apac:candidate:cand_1|cand_2`; `search(e("python"), k=2)` ranks the python chunk first (score desc); `search(..., key_prefix="acme:apac:candidate:cand_1")` returns only cand_1; `delete_by_key_prefix("…cand_1")` removes cand_1's rows and returns the count; a record whose `embed_version` differs from the pinned one → `add` raises/`ValueError` (drift guard) when the store is non-empty; `current_version()` returns the pinned set.
- [ ] Run → fail; implement `record.py` (dataclass + `new_id()`), `store.py` (SQLite table `vectors(vector_id PK, memory_key, embed_model, embed_version, struct_ref, source_ref, text, embedding TEXT)`; parameterised SQL; `search` loads matching rows, cosine, sort desc, top-k; `delete_by_key_prefix` uses `memory_key LIKE ? || '%'` with the literal prefix escaped, or `memory_key = prefix OR memory_key LIKE prefix||':%'`; drift: on `add`, if store non-empty and record.embed_version ∉ current_version() → raise `ValueError`); run → pass; commit `feat(memory): vector record + SqliteVectorStore (cosine NN, key-prefix cascade, drift guard)`.
