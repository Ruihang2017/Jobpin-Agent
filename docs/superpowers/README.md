# `docs/superpowers/` — specs & plans

## English
Artifacts from the per-point development workflow (see root `CLAUDE.md` §5).

- `specs/` — design specs (the validated design for each point/feature).
- `plans/` — implementation plans (bite-sized, testable tasks derived from a spec).

Each point produces one spec and one plan, dated `YYYY-MM-DD-<topic>`.

## 中文
每节点开发工作流的制品（见根 `CLAUDE.md` §5）。

- `specs/` — 设计规格（每个节点/功能经确认的设计）。
- `plans/` — 实现计划（由规格派生的、可测试的小颗粒任务）。

每个节点产出一份规格与一份计划，以 `YYYY-MM-DD-<主题>` 命名。
