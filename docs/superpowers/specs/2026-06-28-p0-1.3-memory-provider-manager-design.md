## 1. Goal

Port Hermes's pluggable-memory **abstract interface (`MemoryProvider`)** and **lifecycle orchestrator
(`MemoryManager`)** + the `<memory-context>` **fence helpers**, then add a **minimal built-in provider**
wrapping the §1.2 `MemoryStore`. The result attaches to the §1.1 agent loop **through the existing
`MemoryHooks` seam — with no change to `agent_loop.py`** — so the curated Org/Recruiter snapshot reaches
the system prompt and the prefetch/sync/compression/session-switch lifecycle is in place for the
large-volume entity providers (§1.4) and governance (§1.5) to plug into.

This is the contract layer of the Memory Subsystem (PRD §9.2/§9.3): the small-volume curated store
(§1.2) and the future large-volume retrieval stores (§1.4) both hide behind one `MemoryProvider`
interface, indistinguishable to the loop.

## 5. The linchpin — `MemoryManagerHooks` adapter (attaches with NO loop change)

The §1.1 loop calls `self.hooks.prefetch(...)` / `after_turn(...)` / `on_delegation(...)` /
`on_session_switch(...)` / `on_pre_compress(...)` (the `MemoryHooks` Protocol in `core/hooks.py`). The
ported Manager speaks a different vocabulary (`prefetch_all` / `sync_all` / `queue_prefetch_all` / …).
`MemoryManagerHooks` bridges the two:

```
class MemoryManagerHooks:                      # implements core.hooks.MemoryHooks (duck-typed)
    def __init__(self, manager): self._m = manager
    def prefetch(self, query, session_id) -> str:
        return build_memory_context_inner(self._m.prefetch_all(query, session_id=session_id))
    def after_turn(self, session_id, messages) -> None:
        user, assistant = _last_user_text(messages), _last_assistant_text(messages)
        self._m.sync_all(user, assistant, session_id=session_id, messages=_as_dicts(messages))
        self._m.queue_prefetch_all(user, session_id=session_id)
    def on_delegation(self, task, result, child_session_id) -> None:
        self._m.on_delegation(task, result, child_session_id=child_session_id)
    def on_session_switch(self, new, parent, reset, rewound) -> None:
        self._m.on_session_switch(new, parent_session_id=parent or "", reset=reset, rewound=rewound)
    def on_pre_compress(self, messages) -> str:
        return self._m.on_pre_compress(_as_dicts(messages))
```

- `_last_user_text` / `_last_assistant_text` pull the last USER / ASSISTANT content from the §1.1
  `Message` history that `after_turn` receives; `_as_dicts` renders `Message`s to the OpenAI-style dicts
  the provider contract expects.
- `after_turn` returns immediately — `sync_all`/`queue_prefetch_all` dispatch to the background worker;
  tests assert persistence via `manager.flush_pending(timeout)`.

## 6. Fence ownership (loop owns the outer tags)

`agent_loop.py` already wraps recall as `f"<memory-context>\n{recall}\n</memory-context>"`. So the
adapter's `prefetch()` must return the **inner** content (system note + sanitized recall, no outer
tags); the loop adds the outer fence, reproducing Hermes's block exactly. `fence.py`:

```
def sanitize_context(text): ...                       # ported: strip fence tags / internal blocks / notes
def build_memory_context_inner(raw):                  # NEW (§1.1-seam adaptation)
    if not raw or not raw.strip(): return ""
    return _SYSTEM_NOTE + "\n\n" + sanitize_context(raw)
def build_memory_context_block(raw):                  # ported full block, defined via the inner
    inner = build_memory_context_inner(raw)
    return "" if not inner else f"<memory-context>\n{inner}\n</memory-context>"
```

`_SYSTEM_NOTE` is Hermes's exact "[System note: … NOT new user input … authoritative reference data …]"
string. Tested: `<memory-context>\n` + `build_memory_context_inner(x)` + `\n</memory-context>` ==
`build_memory_context_block(x)`.

## 7. The built-in provider (`BuiltinMemoryProvider`)

Wraps the §1.2 store; `name == "builtin"` (always registered first). In §1.3 it is deliberately lean —
the curated snapshot reaches the prompt **directly via the `memory_snapshot` slot** (Plan §1.1 assembly
order), so the provider does **not** re-inject it:

| Method | §1.3 behaviour | Why |
|---|---|---|
| `system_prompt_block()` | `""` | the snapshot goes to the `memory_snapshot` slot directly from the store; provider block is for *static* info — returning the snapshot here would duplicate it |
| `prefetch()` | `""` | curated memory is static in the prompt; per-query recall is §1.4 |
| `sync_turn()` | no-op | curated memory is hand-edited, not auto-written per turn (writes are explicit; the governed tool is §1.5) |
| `get_tool_schemas()` | `[]` | the governed `memory` write tool is §1.5 |
| `on_pre_compress()` | `""` | the **seam §1.6 needs** ("MemoryStore is not a Provider" gap, Plan §1.6); real extraction is §1.6 |
| `is_available()` / `initialize()` / `shutdown()` | `True` / no-op (store already loaded) / no-op | local, no resources |

Its value in §1.3: it makes the file store a Provider (the §1.6 seam) and exercises the Manager
lifecycle. The dynamic paths (recall, governed writes) light up in §1.4/§1.5.

## 8. Lifecycle & threading (ported invariants)

- **Serial background persistence (Key Invariant #5):** `sync_all`/`queue_prefetch_all` submit to a
  lazily-created `ThreadPoolExecutor(max_workers=1, thread_name_prefix="mem-sync")` — one worker
  guarantees turn N persists before N+1; the main turn never blocks. Retain `max_workers=1` + the name.
- **`flush_pending(timeout)`:** submit a sentinel to the single worker and wait — a deterministic barrier
  used at session boundaries and by tests.
- **Bounded drain on shutdown:** `shutdown_all` → `_drain_sync_executor` cancels queued work, waits at
  most `_SYNC_DRAIN_TIMEOUT_S` on a daemon watcher; a wedged provider dies with the interpreter, never
  hanging exit.
- **Failure isolation:** every provider hook call is wrapped in try/except + `logger.warning|debug`;
  one provider's failure blocks neither the others nor the turn.
- **Single external provider:** `add_provider` accepts `builtin` always; a second non-builtin is rejected
  with a warning (`CompositeMemoryProvider` is Phase 2). **Routing seam reserved**, merging not enabled.
- **Shadow guard:** a provider tool whose name is in `_CORE_TOOL_NAMES` is dropped at registration and
  never advertised (built-ins always win).

## 9. Testing & acceptance (= Plan §1.3 matrix + Exit Criteria)

`test_memory_fence.py`: `sanitize_context` strips tags/blocks/notes; inner⊕outer == full block; empty → "".

`test_memory_provider.py`: builtin provider contract (snapshot reaches prompt via `memory_snapshot`;
`system_prompt_block`/`prefetch`/`sync_turn`/`get_tool_schemas`/`on_pre_compress` lean defaults);
ABC enforces the abstract methods.

`test_memory_manager.py` (fake providers):
- **Serial background persistence** — two consecutive `sync_all` calls land in order on the single worker;
  the call returns without blocking.
- **flush barrier visible** — state asserted only after `flush_pending`.
- **Wedged provider doesn't block exit** — a provider whose `sync_turn` sleeps "hundreds of seconds";
  the turn isn't blocked and `shutdown_all` returns within `_SYNC_DRAIN_TIMEOUT_S`.
- **Failure isolation** — a provider hook raising still lets others + the turn proceed.
- **Fence enforced / stripping** — `prefetch_all` output wrapped via the fence; a provider that returns
  pre-wrapped `<memory-context>` is stripped by `sanitize_context`.
- **Second external rejected**; **core tool not shadowed** (a provider tool named `delegate_task`/`clarify`
  is dropped).

`test_memory_manager_hooks.py`: the adapter delegates (`prefetch`→`prefetch_all` wrapped inner;
`after_turn`→`sync_all`+`queue_prefetch_all`; the rest delegate); **end-to-end** — build a memory
backend over a store seeded with an Org standard, wire a real §1.1 `Agent` (`FakeProvider`) with
`hooks=backend.hooks`, `parts.memory_snapshot=backend.memory_snapshot()`; assert the composed system
prompt contains the Org standard and that a turn fires `prefetch`/`after_turn` (persistence visible after
`flush_pending`).

**Exit (Plan §1.3):** builtin provider completes prefetch→turn→sync→queue_prefetch through the Manager
with `flush_pending` visibility; a wedged provider still exits within `_SYNC_DRAIN_TIMEOUT_S`;
`<memory-context>` content is always fenced and provider-included fences are stripped. All offline; the
real-OpenAI path stays opt-in.

## 11. Risks

- **Faithful-port drift** — re-implementing the executor/drain/fence subtly differently. Mitigation:
  port method-by-method against the real source; cite symbols; mirror Hermes's lifecycle cases in tests.
- **Background-thread test flakiness** — timing assertions on a real thread pool. Mitigation: assert via
  `flush_pending` (deterministic barrier), not sleeps; the wedged-provider test bounds on
  `_SYNC_DRAIN_TIMEOUT_S` and uses a daemon worker.
- **Fence double-wrap** — if the adapter returned the full block, the §1.1 loop would nest tags.
  Mitigation: the adapter returns the **inner** block (§6); a test pins inner⊕outer == full block.
- **Scope creep into §1.5/§1.6** — the write tool, agent-surface injection, streaming scrubber, and real
  compression wiring are explicitly **out** (§2); the mechanism is ported and fake-provider-tested, ready
  for those points.
- **Snapshot duplication** — injecting the snapshot both via the store-direct `memory_snapshot` slot and
  the provider block. Mitigation: builtin `system_prompt_block()` returns `""` (§7).
