# Phase 0 §1.10 — Integration framework (connector SDK + anti-corruption layer + MCP skeleton + local-first switch): design spec

**Date:** 2026-06-30
**Workstream:** Production Plan §1.10 (Integration framework)
**Branch:** `phase0/1.10-integration` (off `phase0/1.9-security-baseline`)
**Cycle shape:** one cohesive cycle delivering the integration **framework** + a **fake** read-only ATS
connector; the **live** real-ATS connection is deferred (committed in the §1.10 Plan scope decision).

---

## 1. Goal

Deliver, standalone (no `core/agent_loop.py` change) and with the local-first posture enforced by default:

1. A **connector SDK + anti-corruption layer** that translates external records → §1.8 canonical entities,
   isolating external schema drift (an external field rename touches only the ACL, never the §1.8 schema).
2. An **MCP tool-exposure skeleton** — connector operations exposed as `ToolSpec`s registered in the
   existing `ToolRegistry` (no live MCP server/transport — that is the §1.12 spike).
3. A **`FakeATSConnector`** (read-only, over fixtures) + contract tests proving the pipeline external →
   anti-corruption → canonical → local store, with **no PII and no network**.
4. A **"fully local" switch** (a hard outbound chokepoint; **default on** ⇒ 0 outbound) + **per-egress
   audit** via the §1.8 `AuditStore` (`action="egress"`).

**Exit criteria:** (1) read-only pull (fake) → anti-corruption → canonical entities → local store,
switchable off throughout; (2) fully-local switch on ⇒ 0 outbound calls (dedicated test).

---

## 2. Scope

**In scope (Phase 0):** `integration/sdk.py`, `integration/outbound.py`, `integration/service.py`,
`integration/connectors/fake_ats.py`, `integration/mcp.py`, tests.

**Out of scope (deferred, per the committed scope):** the **live** real-ATS/HRIS connection (trigger: real
credentials **+** the §1.11 de-id pipeline); a live MCP **server/transport** (→ §1.12 spike 5); bidirectional
sync / multiple connectors (Phase 1–2); the real **de-identification** of egress payloads (→ §1.11 — §1.10
records a `deid_status` seam only); OAuth credential flows (deferred with the live connector).

**Invariants preserved:** no `core/agent_loop.py` change; §1.8 `data/` and the `core/memory/governance/
orchestration/security` packages unchanged (the §1.8 `AuditStore` is *used*, not modified — the egress
field-set + de-id status are encoded into the audit `reason`, so no `audit_log` migration this cycle).

## 3. Architecture & components

New package `agent/src/jobpin_agent/integration/`.

### 3.1 `integration/sdk.py` — the connector contract + anti-corruption base

```
@dataclass(frozen=True)
class ExternalRecord:
    source: str                 # connector name, e.g. "fake-ats"
    kind: str                   # "candidate" | "job" | "application"
    raw: dict                   # the opaque external row (external field names)

class Connector(ABC):
    name: str                                       # stable connector id
    def fetch(self, kind: str) -> list[ExternalRecord]   # the (conceptually outbound) read-only pull

class AntiCorruptionLayer(ABC):
    # external record -> §1.8 canonical entity; subclass maps external field names to canonical fields.
    def translate(self, rec: ExternalRecord) -> Candidate | Job | Application   # dispatch by rec.kind
    def _to_candidate(self, raw: dict) -> Candidate     # subclass implements the per-kind mapping
    def _to_job(self, raw: dict) -> Job
    def _to_application(self, raw: dict) -> Application
```

`translate` dispatches on `rec.kind` and raises `ValueError` on an unknown kind (fail closed). The ACL is
the **only** place external field names appear; canonical entities are always well-formed §1.8 dataclasses.

### 3.2 `integration/outbound.py` — the "fully local" switch + egress audit (the chokepoint)

```
class OutboundBlocked(Exception): ...

class OutboundGuard:
    def __init__(self, *, fully_local: bool = True, audit: AuditStore | None = None, actor: str = "system")
    def send(self, *, target: str, fields: list[str], reason: str, deid_status: str = "none",
             call: Callable[[], T]) -> T
        # fully_local=True  -> record action=egress result="rejected:fully_local" (if audit), then RAISE
        #                      OutboundBlocked; `call` is NEVER invoked (0 outbound).
        # fully_local=False -> record action=egress result="ok"; invoke + return call().
```

- **`fully_local` defaults to True** — the safe local-first posture: nothing leaves the machine unless an
  operator explicitly turns the switch off.
- `send` is the single outbound chokepoint: real network I/O happens **only** inside `call`, which only runs
  when the switch is off. So "switch on ⇒ 0 outbound" is structurally guaranteed (the test uses a spy `call`).
- The egress audit row uses the §1.8 `AuditStore.record(actor, "egress", target_key=target, reason=…,
  result=…)`. The **field set + de-id status are encoded into `reason`** (e.g.
  `"pull:candidate fields=[id,name,skills] deid=none"`) — no `audit_log` schema change. `deid_status` is a
  seam: §1.11's `deid` will set it to the real masking status before any real-PII egress.

The guard wraps the fetch, so a fully-local run raises `OutboundBlocked` before any fetch (0 outbound, 0
ingested); a switch-off run logs one egress row, fetches, translates each record through the ACL, and upserts
into the §1.8 `CanonicalStore`.

The fixtures use deliberately **different** field names from the canonical schema (e.g. `full_name` →

### 3.5 `integration/mcp.py` — MCP tool-exposure skeleton

```
def connector_toolspecs(service: IntegrationService, connector: Connector,
                        acl: AntiCorruptionLayer, kinds: list[str]) -> list[ToolSpec]
    # one ToolSpec per (connector, kind): name=f"{connector.name}_pull_{kind}",
    #   description, parameters={"type":"object","properties":{}}, handler=lambda args: <ingest + summary str>

def register_connector(registry: ToolRegistry, *toolspecs) -> None   # convenience
```

Each `ToolSpec` (the MCP tool shape: name / description / JSON-schema params / handler) wraps
`service.ingest(...)` and returns a short string summary. Registering them into a `ToolRegistry` is the
"exposed as MCP tools" deliverable; a live MCP server/transport is the §1.12 spike. The handler respects the
switch (a fully-local handler call returns a "blocked: fully-local" string rather than raising into the loop).

## 4. Data flow

```
external system ──fetch(kind)──▶ ExternalRecord(raw)            [only runs when switch OFF]
       │                                   │
   OutboundGuard.send (switch + egress audit row via §1.8 AuditStore)
       │ switch ON ⇒ raise OutboundBlocked (0 outbound)
       ▼ switch OFF
   AntiCorruptionLayer.translate(rec) ──▶ §1.8 Candidate/Job/Application
       ▼
   CanonicalStore.upsert_* (local store; encrypted at rest if §1.9 keyed)
```

MCP path: agent/tool-caller → `ToolRegistry.execute("fake-ats_pull_candidate")` → handler →
`IntegrationService.ingest(...)` (same flow).

## 5. Error handling

- **Fully-local (default):** `guard.send` raises `OutboundBlocked` and `call` is never invoked — 0 outbound.
  The MCP handler catches it and returns a `"blocked: fully-local mode"` string (never crashes the loop).
- **Unknown kind:** `AntiCorruptionLayer.translate` raises `ValueError` (fail closed).
- **Connector fetch failure** (switch off): propagates; the egress audit row is written *before* the call
  (so a failed egress still leaves a trace), `result` reflects the attempt.
- **No PII outbound without de-id:** structurally, no real connector ships this cycle; `deid_status` defaults
  to `"none"` and the fake uses synthetic data only.

## 6. Testing & acceptance

`agent/tests/integration/`:
- `test_anti_corruption` — the fake ATS's external field names translate to the right §1.8 entity fields;
  unknown kind raises.
- `test_outbound_guard` — fully-local ⇒ `send` raises `OutboundBlocked`, `call` spy invoked **0** times, an
  `egress`/`rejected:fully_local` audit row written; switch-off ⇒ `call` runs once + an `egress`/`ok` row.
- `test_ingest_pipeline` (**exit 1**) — switch off: `ingest` pulls the fake → translates → the
  `CanonicalStore` returns the candidates with canonical field names.
- `test_fully_local_zero_outbound` (**exit 2**) — with the default switch on, `ingest` raises and the fetch
  spy count is 0 (dedicated 0-outbound test).
- `test_mcp_exposure` — `connector_toolspecs` yields registrable `ToolSpec`s; `ToolRegistry.execute` runs the
  ingest handler (switch off) and returns a summary; fully-local handler returns the blocked string.

Full suite stays green (target: 240 passed + the new §1.10 tests, 2 skipped). No `agent_loop.py` change
(git-verified); §1.8/§1.9 packages unchanged.

## 8. Deferrals (explicit)

## 9. Global constraints

- Python ≥ 3.11; `python -m pytest` from `agent/`; stdlib-only (no new dependency — the integration layer
  is pure Python over §1.8/§1.1 primitives).
- TDD; bilingual docstrings (EN block, blank line, 中文 block; never a literal triple-quote inside).
- Local-first default: `fully_local=True`. No PII outbound without de-id. No `agent_loop.py` change.
- Anti-corruption is the only translation point; canonical entities are always §1.8 dataclasses.
