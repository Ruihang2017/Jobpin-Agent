## 1. Goal

Implement the **large-volume, retrieval-needing** entity-semantic memory layer (PRD §9.3): vectorise
candidates / semantic chunks, store + retrieve **locally**, and present it **upward through the same
§1.3 `MemoryProvider` interface** — indistinguishable to the loop from the curated file store. Land
`Candidate` + `Semantic` providers (needed for M1), a **minimal `CompositeMemoryProvider`** that lets
both coexist under the Manager's single-external rule, a **re-embed migration tool**, and a **recall +
P95 benchmark scaffold**.

## 2. Scope (all of §1.4 this cycle; deferrals behind seams)

**In:** `memory/vector/` (record + store ABC + SQLite reference impl + re-embed tool), `memory/structured.py`
(minimal candidate store), `memory/embedding.py` (embed seam + default), `memory/providers/{semantic,
candidate,composite}.py`, `memory/benchmark.py` + a demo, tests, docs, security review.

**Deferred behind seams (decided in brainstorming):**
- **Real vector backend** (sqlite-vec / LanceDB / Chroma) → post-**§1.12** spike; swaps in behind the
  `VectorStore` ABC. §1.4 ships a **stdlib SQLite + brute-force cosine** reference impl.
- **Real embedding model** (BGE-local / OpenAI-embeddings) → config; §1.4 ships a deterministic
  **hashing bag-of-words** `EmbedFn` (real *lexical* similarity, offline, dependency-free).
- **Governance on writes** (provenance/consent labels, reject-unlabelled) → **§1.5**; §1.4 ingest goes
  through a pass-through **`write_gate`** seam (same discipline as the §1.3 write-tool deferral — no
  ungoverned write path opened early).
- **RBAC policy** → **§1.5**; §1.4 ships a **namespace-filter** seam (`scope_filter`, default open) that
  runs **before** vector NN (so §1.5 plugs in "filter then retrieve", not "retrieve then filter").
- **Full Composite** (Employee sub-provider, `entity_type`+intent routing table, merge-consistency
  matrix, `backup_paths` aggregation) → **Phase 2 §3.2**.
- **Full canonical data model** → **§1.8**; §1.4's structured store carries only retrieval/filter fields.

## 4. The vector record + store

```
VectorRecord:
  vector_id: str          # uuid (caller- or store-assigned)
  memory_key: str         # tenant:org:entity_type:entity_id  (RBAC + erasure cascade anchor, §1.0)
  embed_model: str        # embedder name (pinned)
  embed_version: str      # name+dim signature; mismatch => drift => re-embed (no mixed retrieval)
  struct_ref: str         # pointer to the structured row primary key (provenance back-link)
  source_ref: str         # pointer to the original-text chunk (the "back to source" citation)
  text: str               # the chunk text (kept for citation rendering; small at this scale)
  embedding: list[float]  # the vector
```

`VectorStore` (ABC):
```
add(records: list[VectorRecord]) -> None
delete(vector_ids: list[str]) -> int
delete_by_key_prefix(prefix: str) -> int           # erasure cascade (e.g. "acme:apac:candidate:cand_7f3a")
search(query: list[float], k: int, *, key_prefix: str | None = None) -> list[Hit]   # Hit = (record, score)
current_version() -> set[str]                       # distinct embed_versions on disk (drift detection)
```
`SqliteVectorStore(db_path)`: one SQLite table; `search` = brute-force cosine over rows matching the
optional `key_prefix` (the namespace pre-filter); embeddings stored as JSON/blob. **Rejects a record
whose `embed_version` differs from the store's pinned version** (no silent mixing) unless the store is
empty (first write pins it). At hundreds–thousands of rows, brute-force cosine is well under the P95
target; the ABC lets sqlite-vec/LanceDB replace it after §1.12.

## 5. Embedding seam (`embedding.py`)

`EmbedFn = Callable[[str], list[float]]`. Default `hashing_embedder(dim=256)`: lowercase-tokenise →
hash each token mod `dim` → count → L2-normalise. Deterministic, dependency-free, and captures **lexical
overlap** (so "Python engineer" ~ "python developer" share a dim) — enough for the recall demo + tests.
`embed_version(name, dim) -> "name@dim"`. Real semantic embedders (BGE via sentence-transformers, or
OpenAI `text-embedding-3`) are config-injected later behind the same `EmbedFn` type — a *model swap*
triggers the re-embed migration (§7).

## 6. The entity providers

Both implement the §1.3 `MemoryProvider` contract; both keep `prefetch` fast and citation-bearing.

- **`SemanticRAGProvider`** (`name="semantic"`, `entity_type="semantic"`): a knowledge-base RAG index.
  `ingest(doc_id, text, *, memory_key, source_ref)` embeds + stores. `prefetch(query)` = embed query →
  `scope_filter` → vector NN → rerank → fenced text with `[source: <source_ref>]` citations. `sync_turn`
  no-op (read-only retrieval; KB updates are explicit `ingest`). 
- **`CandidateMemoryProvider`** (`name="candidate"`, `entity_type="candidate"`): candidate profiles.
  `ingest(candidate)` writes the **structured row** (via `write_gate`) **and** the derived **vectors**
  (resume chunks). `prefetch(query)` = structured filter (skills/location/work-rights via `scope_filter`)
  **then** vector NN over the surviving `memory_key` prefixes → citations back to the candidate +
  chunk. `delete(memory_key)` cascades: structured row + `delete_by_key_prefix` on the vector store
  (the §1.5 erasure mechanism). `sync_turn` minimal (the recruiter adopt/override feedback loop is M1/Phase 1).

**Fast prefetch (Plan):** `queue_prefetch(query)` runs the NN+rerank in the §1.3 background worker and
**caches** the result by `(query, session_id)`; `prefetch` returns the cache, falling back to a
synchronous compute when cold (bounded — fine at this scale). Returned text is raw recall (the §1.3
fence wraps it).

## 7. Re-embed migration (`vector/reembed.py`)

`reembed(store, new_embed_fn, new_version, get_text, *, checkpoint) -> ReembedResult`. Detects drift
(`store.current_version()` ≠ `new_version`), then for each record: recompute the embedding from its
`text` (or `get_text(source_ref)`), write to a **new** store/table, validate (count + a sample
round-trip), then **atomically switch**. **Resumable:** a checkpoint records the last migrated
`vector_id`; re-running skips done records. Mixed-version retrieval is **forbidden** mid-migration
(the store rejects a query whose pinned version is mid-switch). Tested by switching `dim` and asserting
resume-after-interrupt.

## 8. Minimal `CompositeMemoryProvider` (`providers/composite.py`)

`name="composite"` — the **sole external** provider (builtin still first; Manager `add_provider`
unchanged). Holds an ordered list of sub-providers (Semantic, Candidate).
- `prefetch(query, session_id)`: call each sub-provider's `prefetch` → concatenate → split on
  `ENTRY_DELIMITER` → drop blanks → `list(dict.fromkeys(...))` (order-preserving dedup) → truncate to a
  char budget → return (the Manager/§1.3 fence wraps it).
- `queue_prefetch`: fan out to each sub-provider (still on the single worker via the Manager).
- `sync_turn`: **unicast** to the owning sub-provider by a simple route (a `route(entity_type)` map;
  §1.4's sync is mostly inert, but the unicast mechanism is in place); skip when `agent_context` is
  non-primary.
- `system_prompt_block` / `on_pre_compress` / `on_session_switch` / `on_delegation` / `on_memory_write` /
  `backup_paths` / `shutdown`: fan out (reverse order for shutdown). `get_tool_schemas` aggregates (none yet).
- It does **not** touch the Manager's single-worker / `flush_pending` / drain — those invariants hold.

## 9. Seams (injected; default-safe; real impls later)

| Seam | §1.4 default | Real impl |
|---|---|---|

## 10. Testing & acceptance (= Plan §1.4 Exit Criteria + matrix)

- `test_vector_store`: add/search cosine ranking; `delete_by_key_prefix` cascade; **drift guard** (a
  record with a foreign `embed_version` is rejected); `key_prefix` pre-filter.
- `test_embedding`: deterministic; lexical overlap → non-zero cosine; `embed_version` signature.
- `test_structured_store`: upsert/get/filter/delete-by-prefix.
- `test_semantic_provider` / `test_candidate_provider`: **ingest → NL query recalls the right item with a
  `[source: …]` citation**; structured pre-filter; `prefetch` fast-path (cache via `queue_prefetch`);
  `delete` cascades structured + vectors; `write_gate` holds an ungated ingest.
- `test_composite`: broadcast prefetch merges + dedups two sub-providers; unicast sync routes to the
  owner; registered as the sole external on a real `MemoryManager` (second external still rejected);
  fan-out hooks; reverse `shutdown`.
- `test_reembed`: version switch → full re-embed → validate; **resume after interrupt** skips done records.
- `test_benchmark`: `run_recall_benchmark` returns `recall@k` + `p95_ms` over a seeded set.
- **End-to-end** (`test_recall_demo` + `examples/recall_demo.py`): parse N résumés → ingest → wire the
  Composite as the sole external on a `MemoryManager` + the §1.3 hooks + a real §1.1 `Agent`
  (`FakeProvider`) → an NL query's recall (with citations) reaches the model as a `<memory-context>`
  message — **offline, no loop change**.

**Exit (Plan §1.4):** résumé batch → vectorise → NL recall **with provenance citation** ✅; re-embed
migration after a version switch, **resumable** ✅; recall **P95** at hundreds–thousands via the
benchmark ✅ (with the stdlib backend + fake embedder; real backend/embedder validated at §1.12).

## 11. Security review, triple-review, documentation

- **Security review** (`docs/security/p0-1.4-…-review.md`): SQL built with parameterised queries (no
  injection); `memory_key` prefix matching is exact-prefix (no wildcard injection); embeddings are
  numeric (no code path); `write_gate`/`scope_filter` default-safe and documented; deletion cascade is
  bounded; no `eval`/`exec`; the fake embedder is not a security control (semantic scan is §1.6); the
  "retrieve-then-filter" leak is closed by filtering **before** NN.
- **Triple-review** (senior engineer / architect / PM) vs the Plan; **fix the Plan first** if wrong.
- **Bilingual devlog** `site/devlog/p0-1.4-vector-entity-providers{,-EN}.md` (how + why; a diagram of
  ingest + retrieval + the Composite-under-Manager wiring), per `CLAUDE.md` §6; update notices, folder
  guide, devlog index, `CLAUDE.md` §8.

## 12. Risks

- **Scope (large cycle)** — mitigate by building/committing each layer independently TDD'd (vector →
  structured → embed → providers → composite → reembed → benchmark), reviewed as a whole.
- **Fake embedder ≠ semantic recall** — lexical-overlap only; documented; the recall *demo* shows the
  pipeline, real semantic quality is a config swap + the §1.12 benchmark. No correctness claim rests on it.
- **Brute-force cosine scaling** — fine at hundreds–thousands (the stated scale); the ABC makes the
  production backend a swap; the benchmark surfaces the ceiling.
- **Composite-forward = building ahead?** — no: the ≥2-provider trigger fires at §1.4; the Plan is
  corrected to say so; only the *minimal* Composite is built, the full one stays §3.2.
- **Ungoverned writes before §1.5** — closed by the `write_gate` seam (pass-through now, real gate §1.5),
  exactly as §1.3 deferred its write tool.
