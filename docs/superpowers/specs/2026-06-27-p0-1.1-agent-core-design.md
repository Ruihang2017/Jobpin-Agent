## 1. Goal

A self-contained, **provider-agnostic, locally-running agent core** that completes one full turn —
**system-prompt assembly → tool-call loop → sub-agent delegation** — plus a **minimal OpenAI
adapter** so the turn runs against a real model. It exposes stable extension points for the Memory
Subsystem (§1.2–1.6) and HR modules to attach later, without the loop needing rework.

This is the foundation point of Phase 0; memory, governance, orchestration, integration, and the
full AI/eval platform attach to it in later points.

## 2. Scope (this point only)

In: the conversation loop, deterministic system-prompt assembly, sub-agent delegation, SQLite session
store, a provider-agnostic `ModelProvider` abstraction + a minimal OpenAI adapter + a deterministic
fake provider, a minimal tool registry with one demo tool, lightweight step-level tracing, env-based
config, a runnable demo, the provenance/NOTICE update, and the bilingual study doc.

Out (YAGNI — later points): the Memory Subsystem (§1.2–1.4), HR memory governance (§1.5), the
injection-defence port + real pre-compression fact injection (§1.6), Layer B orchestration (§1.7),
canonical data model/audit (§1.8), security baseline (§1.9), integration/MCP (§1.10), the full model
router/de-id/eval/tracing platform and the Claude/DeepSeek/local adapters (§1.11), streaming.

## 3. Grounding & build strategy (Tenet 1 of `TEXTBOOK_SPEC.md`)

§1.1 is a **rewrite that borrows design**, not a code port (the Hermes loop is coupled to its
CLI/TUI/gateway/multi-provider pipeline; we need local-first + clean ownership). We borrow the turn
concept from `run_conversation`, the assembly approach from `build_system_prompt*`, the delegation
invariants from the `on_delegation` pattern, and the pre-compression hook *signature* from
`conversation_compression.py`. No substantial Hermes code is copied this point; the
`THIRD_PARTY_NOTICES.md` provenance table records each core file as **design-derived (new/adapt)**.
Code-porting (the memory subsystem) begins at §1.2.

## 4. Components (under `agent/src/jobpin_agent/`)

Each file has one responsibility and a tested interface.

- `core/messages.py` — provider-agnostic types: `Role`, `Message`, `ToolCall`, `ToolResult`,
  `ModelResponse` (either `text: str` or `tool_calls: list[ToolCall]`). No provider wire format here.
- `core/model/provider.py` — `ModelProvider` ABC: `complete(messages: list[Message], tools:
  list[ToolSpec] | None) -> ModelResponse`.
- `core/model/fake_provider.py` — `FakeProvider(script: list[ModelResponse])`: returns scripted
  responses in order; records the calls it received. Deterministic, offline.
- `core/model/openai_provider.py` — `OpenAIProvider(config)`: maps `Message`/`ToolSpec` ↔ OpenAI
  Chat Completions, returns a `ModelResponse`. Key from `OPENAI_API_KEY`; model id from config.
- `core/tools.py` — `ToolSpec` (name, description, JSON-schema params), `ToolRegistry`
  (register/lookup/execute), and one demo tool `echo(text)` to exercise the tool path.
- `core/system_prompt.py` — `build_system_prompt(parts: SystemPromptParts) -> str` with a **fixed,
  deterministic order**: org policy / compliance / role permissions → memory snapshot slot (empty)
  → provider static block slot (empty) → tool descriptions. Same input → byte-identical output.
- `core/tracing.py` — `Tracer` recording ordered step events (`model_call`, `tool_call`,
  `delegation`) with timestamps; in-memory + JSONL sink. Timestamps injected (no `Date.now()` purity
  concern here, but the tracer takes a `clock` callable for testability).
- `core/agent_loop.py` — `Agent(provider, tools, session_store, tracer, hooks)` with
  `run_turn(session_id, user_input) -> TurnResult`. Flow: `hooks.prefetch(query)` (default "") →
  `build_system_prompt` → `provider.complete` → if `tool_calls`: execute via registry, append
  `ToolResult`, loop → else final answer → `hooks.after_turn(messages)` (default no-op) → persist.
  **Stop condition**: `max_tool_iterations` (default 8) → returns a `stopped` TurnResult.
- `core/delegation.py` — `delegate(parent, task) -> DelegationResult`: runs a child `Agent.run_turn`
  with `skip_memory=True` (child gets no memory hooks); parent observes via
  `hooks.on_delegation(task, result, child_session_id)` (default no-op).
- `core/session_store.py` — `SessionStore(db_path)` over SQLite (single file): `create_session`,
  `append_message`, `get_messages`, `branch(session_id) -> new_id`, `reset(session_id)`; each of
  branch/reset fires `hooks.on_session_switch(new, parent, reset, rewound)`.
- `core/config.py` — `CoreConfig.from_env()`: `openai_api_key`, `model_id` (configurable default),
  `db_path`, `max_tool_iterations`. Secrets only from env; never written to disk/git.
- `core/hooks.py` — `MemoryHooks` protocol with default no-op implementation (`prefetch`,
  `after_turn`, `on_delegation`, `on_session_switch`, `on_pre_compress`). This is the seam §1.2–1.6
  fill in. The loop depends only on this protocol.
- `examples/demo_turn.py` — runs plain-text, tool-call, and delegation turns end-to-end with traces.

**Dependencies:** add `openai` (runtime) and keep `pytest` (dev) in `agent/pyproject.toml`.

## 5. Key decisions

- **Synchronous core.** Request/response loop; matches Hermes's sync-core + background-threads
  pattern (memory's background sync worker arrives in §1.3 as a thread, not by making the loop
  async). Revisitable if streaming/concurrency is needed.
- **Provider-agnostic by construction.** Internal `Message`/`ToolSpec`/`ModelResponse`; adapters map
  to/from wire formats. OpenAI validates the abstraction now; Claude/DeepSeek/local slot in at §1.11
  behind the same ABC.
- **Memory seam is real but no-op.** `core/hooks.py` defines the protocol; defaults do nothing.
  §1.2–1.6 provide real implementations without touching `agent_loop.py`.
- **Determinism for prefix caching.** `build_system_prompt` is pure and order-fixed; locked by a
  golden-snapshot test built 100× (Key Invariant #1).
- **Delegation invariants** (Key Invariant #3): children run `skip_memory=True`; sensitive writes
  are a parent responsibility (no-op until memory exists).
- **Compression** is an exposed hook signature only; real pre-compression fact injection is §1.6.

## 6. Data flow (one turn)

```
user_input
  → hooks.prefetch(query)            # "" for now (memory recall later)
  → build_system_prompt(parts)       # org/role + empty memory/provider slots + tool descriptions
  → provider.complete(messages,tools)
       ├─ tool_calls? → registry.execute → append ToolResult → loop (≤ max_tool_iterations)
       └─ final text → done
  → hooks.after_turn(messages)       # no-op for now (memory sync later)
  → session_store.append_message(...)
(every step → tracer.event)
```

## 7. Testing & acceptance (= §1.1 exit criteria)

Unit tests use `FakeProvider` (offline, deterministic):
- **Loop paths**: plain answer; single tool call → answer; multi-turn tool continuation (tool →
  model → tool → answer); stop condition (script never finishes → `stopped` after
  `max_tool_iterations`).
- **System-prompt golden snapshot**: assert the assembler output equals a committed golden file, and
  building it 100× yields byte-identical output (prefix stability).
- **Session store**: create/append/get round-trip; `branch` forks history; `reset` clears; the
  `on_session_switch` hook fires with correct flags.
- **Delegation**: child runs with `skip_memory=True` (no memory hooks invoked on child); parent
  `on_delegation` observation fires with the child result.
- **Tracing**: a turn with one tool call produces ordered `model_call`/`tool_call` events.
- **OpenAI adapter**: mapping unit tests (Message/ToolSpec ↔ OpenAI payloads) with a mocked client;
  plus an **opt-in integration test** marked to **skip unless `OPENAI_API_KEY` is set** (real turn,
  no CI dependency on network/cost).
- **Demo**: `examples/demo_turn.py` runs plain + tool + delegation locally, emitting traces.

Acceptance (per Plan §1.1): the end-to-end sequence runs locally with step-level tracing; the
golden-snapshot test passes (100× identical); the provenance table covers every core file and the
MIT notice is in `THIRD_PARTY_NOTICES.md`.

## 9. Risks

- **Misjudging the rewrite/port boundary** → keep the loop a rewrite; provenance table reviews each
  file; cite real Hermes symbols (mitigated by §3).
- **OpenAI API shape drift** (Chat Completions vs Responses API) → isolate all wire mapping inside
  `openai_provider.py`; the rest of the core depends only on `ModelProvider`. Default to Chat
  Completions; swapping is a one-file change.
- **Hidden async need** → if a later point needs streaming, introduce async at the provider boundary
  only; the loop's sync contract stays the seam.
